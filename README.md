# Hermes помощник для пользователей Element

Этот набор файлов — стартовый комплект для отдельного экземпляра Hermes, который отвечает на вопросы пользователей по работе со служебным мессенджером Element/Matrix.

## Что внутри

- `SYSTEM_PROMPT.md` — системный промпт агента.
- `RAG_DESIGN.md` — практическая схема RAG и контуров знаний.
- `knowledge/seed-faq.md` — стартовая база знаний в markdown.
- `knowledge/intent-catalog.yaml` — каталог интентов и ожидаемого поведения.
- `knowledge/chunking-rules.md` — правила подготовки документов для индексации.

## Целевая роль агента

Агент помогает конечным пользователям:
- вход и первичная настройка Element;
- комнаты, уведомления, устройства, ключи, шифрование;
- базовые правила использования рабочего мессенджера;
- куда эскалировать проблему, если нужен администратор.

Агент **не** должен:
- выдумывать настройки, которых нет в документации компании;
- выдавать себя за администратора системы;
- советовать небезопасные действия без предупреждения;
- раскрывать лишнюю внутреннюю информацию об инфраструктуре.

## Минимальный состав источников для RAG

1. Локальная инструкция пользователя по Element.
2. FAQ внедрения служебного мессенджера.
3. Политика использования рабочего мессенджера.
4. Инструкция по входу/SSO/2FA/VPN/мобильным устройствам.
5. Контакты и правила эскалации в ИТ-поддержку.
6. Список разрешённых клиентских настроек и ограничений.

## Быстрый запуск

1. Создать отдельный профиль Hermes для пользовательского помощника.
2. Подключить этот системный промпт.
3. Индексировать документы из `knowledge/` и корпоративной документации.
4. Ограничить доступные toolsets: обычно `file`, `web`/внутренний retriever, возможно `clarify`; без лишних админских инструментов.
5. Протестировать 20–30 типовых пользовательских вопросов.

## Практический совет

Лучше делать не "универсального ИИ по Matrix", а именно помощника по **вашему развёртыванию** Element:
- какой homeserver используется;
- как логинятся сотрудники;
- как устроены комнаты и пространства;
- что делать при потере сессии/ключей;
- какие клиенты разрешены;
- куда писать при инциденте.
