# Hermes Element Helpdesk Assistant

AI-помощник для пользователей служебного мессенджера Element/Matrix.
Развёртывается как изолированный Docker-контейнер с отдельным профилем Hermes.

## Возможности

- Ответы на типовые вопросы пользователей Element (вход, уведомления, комнаты, шифрование, устройства)
- База знаний из 30+ документов с общими инструкциями по Element/Matrix
- Организационные шаблоны (FAQ, политика, эскалация) — нужно заполнить под вашу организацию
- Ограниченный набор инструментов: только чтение KB + memory + skills + session_search
- Самообучение: агент запоминает эффективные формулировки ответов и улучшает их со временем

## Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/dmitreshel-art/hermes_element.git
cd hermes_element
```

### 2. Заполните переменные окружения

Скопируйте пример и заполните реальные значения:

```bash
cp hermes-home/.env.example hermes-home/.env
```

Обязательные переменные:

| Переменная | Описание |
|---|---|
| `MATRIX_HOMESERVER` | URL вашего Matrix-сервера (например `https://matrix.example.org`) |
| `MATRIX_USER_ID` | Matrix ID бота (например `@helpdesk:example.org`) |
| `MATRIX_PASSWORD` или `MATRIX_ACCESS_TOKEN` | Пароль бота или access token |
| `MATRIX_ALLOWED_USERS` | Список разрешённых пользователей (через запятую) или `*` для всех |
| `OPENAI_API_KEY` | Ключ к OpenAI-совместимому API |

### 3. Заполните KB-плейсхолдеры

В файлах базы знаний (`hermes-home/knowledge/`) есть шаблоны вида `ELEMENT_WEB_URL`, `SUPPORT_CONTACT` и т.д.
Замените их на реальные значения вашей организации.

Список всех плейсхолдеров:

```bash
grep -rn 'CHANGE_ME\|ELEMENT_WEB_URL\|SUPPORT_CONTACT\|LOGIN_METHOD\|VPN_REQUIRED\|ALLOWED_CLIENTS' hermes-home/knowledge/
```

### 4. Запустите Docker

```bash
docker compose -f docker/docker-compose.yml up -d
```

### 5. Проверьте

```bash
docker compose -f docker/docker-compose.yml logs -f
```

Бот должен подключиться к Matrix и отвечать на сообщения в разрешённых комнатах/DM.

## Безопасность

### Активные toolsets

| Toolset | Статус | Назначение |
|---|---|---|
| `file` | ✅ Включён | Чтение файлов базы знаний |
| `memory` | ✅ Включён | Память между сессиями |
| `skills` | ✅ Включён | Самообучение (улучшение ответов) |
| `session_search` | ✅ Включён | Поиск по прошлым диалогам |

### Отключённые toolsets

| Toolset | Статус | Почему отключен |
|---|---|---|
| `terminal` | ❌ | Нет доступа к оболочке |
| `browser` | ❌ | Нет доступа к браузеру |
| `web` | ❌ | Нет доступа к вебу |
| `code_execution` | ❌ | Нет выполнения кода |
| `vision` | ❌ | Нет анализа изображений |
| `image_gen` | ❌ | Нет генерации изображений |
| `tts` | ❌ | Нет генерации речи |
| `messaging` | ❌ | Нет отправки сообщений напрямую |
| `homeassistant` | ❌ | Нет доступа к умному дому |
| `todo` | ❌ | Нет управления задачами |
| `clarify` | ❌ | Минимизация интерактивных ветвлений |
| `cronjob` | ❌ | Нет планирования задач |
| `delegation` | ❌ | Нет делегирования подзадач |

Ограничения применены **механически** в `config.yaml`, не только через системный промпт.

## Политика памяти

### Что агент хранит в пользовательской памяти
- Уровень навыков пользователя (новичок / опытный)
- Предпочитаемый клиент (web / desktop / iOS / Android)
- Повторяющиеся затруднения (например, «путается в уведомлениях»)
- Предпочитаемый стиль ответа (подробный / краткий)

### Что агент НЕ хранит
- Пароли, recovery-ключи, токены
- Содержание сообщений
- Персональные данные
- Секреты

### Что агент хранит в skills
- Улучшенные формулировки ответов
- Mini-playbook'и для повторяющихся сценариев
- Безопасные формулировки для сложных тем (ключи, шифрование)

## Кастомизация

### Добавление локальных документов

Положите ваши внутренние документы в `hermes-home/knowledge/`:
- Инструкция по первому входу
- Правила использования служебного мессенджера
- Контакты ИТ-поддержки
- Политика по файлам/звонкам/гостям

### Добавление FAQ

Создайте файл `hermes-home/knowledge/faq-YOUR-TOPIC.md` по шаблону:

```markdown
# FAQ: Название темы

## В: Вопрос?
**А:** Ответ.

## В: Другой вопрос?
**А:** Другой ответ.
```

### Использование другого LLM-провайдера

Отредактируйте `hermes-home/config.yaml` — замените секцию `providers` на нужного провайдера
и укажите API-ключ в `.env`.

## Перенос на другой сервер

1. Клонируйте этот репозиторий на целевой сервер
2. Заполните `.env` реальными значениями
3. Заполните KB-плейсхолдеры
4. `docker compose -f docker/docker-compose.yml up -d`
