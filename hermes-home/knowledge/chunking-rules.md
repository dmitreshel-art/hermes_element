# Правила подготовки chunk'ов для RAG

## 1. Общие правила

- Один chunk = одна практическая тема.
- Не смешивать в одном chunk вход, уведомления и шифрование.
- Не делать chunks слишком длинными.
- Каждый chunk должен быть понятен без чтения всего документа.

## 2. Рекомендуемый размер

- 400–900 слов для обычного how-to.
- 150–300 слов для короткого FAQ/эскалации.

## 3. Обязательные метаданные

В начало каждого документа/куска добавляйте блок:

```yaml
source_id: faq-login-001
title: Первый вход в Element
doc_type: faq
product_area: login
audience: employee
platform: all
criticality: normal
version: 1.0
updated_at: 2026-04-17
authoritative: true
```

## 4. Что отдельно выделять в chunk

Отдельными chunk'ами лучше хранить:
- симптомы ошибки;
- пошаговую инструкцию;
- предупреждение о рисках;
- условия эскалации;
- контакты поддержки.

## 5. Контроль качества

Перед индексацией проверьте:
- нет ли секретов;
- актуальна ли инструкция;
- указана ли целевая платформа;
- ясно ли, что пользователь делает сам, а что делает ИТ;
- нет ли противоречий с действующей политикой.
