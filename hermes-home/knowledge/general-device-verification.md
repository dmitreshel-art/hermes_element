# Общая справка: подтверждение устройств в Element

## Зачем это нужно
Подтверждение устройства нужно, чтобы:
- считать устройство доверенным;
- безопасно читать зашифрованные сообщения;
- использовать доверенное устройство для подтверждения других устройств.

## Как обычно выглядит проверка
В общих сценариях Element предлагает подтвердить новое устройство одним из двух способов:
1. через другое уже подтверждённое устройство;
2. через recovery key / Secure Backup.

## Что безопасно советовать пользователю
Если появилось новое устройство:
1. Сначала проверить, есть ли у вас старое активное доверенное устройство.
2. Если есть — лучше подтверждать через него.
3. Если доверенного устройства нет, использовать recovery key, если он точно сохранён и известен.
4. После подтверждения проверить, открываются ли зашифрованные сообщения.

## Как в общем виде понять, подтверждено ли устройство
В официальной документации Element для web/desktop описан просмотр статуса устройства через раздел Sessions; подтверждённые устройства обычно отмечаются зелёным shield/щитом.

## Когда надо насторожиться
- устройство осталось unverified;
- клиент предлагает рискованный reset вместо нормальной верификации;
- после входа видна учётная запись, но не читается старая зашифрованная история;
- вы не узнаёте часть устройств в списке.

## Чего не стоит делать
- не игнорировать unverified device, если на нём нужно продолжать работу;
- не подтверждать устройства без понимания, какое из них доверенное;
- не сбрасывать recovery / identity только потому, что это самый быстрый путь в интерфейсе.

## Короткий общий ответ пользователю
Если спросили: "что значит подтвердить устройство?"
Можно отвечать так:
- это стандартная процедура доверия между вашими сессиями в Element;
- обычно лучше подтвердить новое устройство со старого доверенного устройства или с помощью recovery key;
- если вы не уверены, какой вариант безопасен, не делайте reset наугад.
