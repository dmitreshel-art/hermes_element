# Правила использования служебного мессенджера

## Кратко
- Служебный мессенджер используется для рабочих коммуникаций.
- Используйте вашу учётную запись и только разрешённые клиенты: Element Web, Desktop, Android, iPhone.
- Внешних пользователей приглашать нельзя.
- Если сообщение с ошибкой ушло не туда, или вы видите риск по безопасности, сразу напишите этому боту.
- Если точных локальных правил по конкретной ситуации пока нет, бот должен честно сказать об этом и дать безопасный общий совет.

## Для чего используется служебный мессенджер
Служебный мессенджер используется для:
- рабочих обсуждений;
- координации по задачам и процессам;
- переписки в комнатах и пространствах по проектам, функциям и командам;
- оперативных уточнений, когда нужен быстрый рабочий контакт.

Не стоит использовать его как место для хранения секретов, уникальных кодов доступа и другой критичной информации, если для этого не предусмотрен отдельный безопасный процесс.

## Учётная запись и устройства
- Для входа используйте вашу учётную запись.
- Разрешённые клиенты: Element Web, Desktop, Android, iPhone.
- Использование личных устройств разрешено.
- При входе с нового устройства может потребоваться подтверждение через старое устройство.
- Если видите незнакомую сессию или не можете подтвердить устройство, сразу напишите этому боту.

## Комнаты и пространства
- Пространство — это группа связанных комнат.
- Комната — это место для конкретной переписки.
- Комнаты и пространства могут создавать все пользователи.
- Перед созданием новой комнаты проверьте, нет ли уже существующей комнаты для той же задачи или процесса.
- Если сомневаетесь, лучше использовать существующую комнату или сначала уточнить через этого бота.

## Временное правило именования комнат
Пока отдельный локальный регламент не утверждён, используйте простой практический принцип:
- название должно быть коротким и понятным;
- из названия должно быть видно назначение комнаты;
- не создавайте несколько комнат с почти одинаковыми названиями для одной и той же темы;
- для временных обсуждений лучше явно указывать тему или проект.

Примеры хороших названий:
- `Проект А — внедрение`
- `ИТ — поддержка пользователей`
- `Закупки — согласование`

Плохая практика:
- слишком общие названия вроде `Работа`, `Общее 2`, `Новая комната`;
- дубли существующих комнат;
- названия, из которых непонятно назначение.

## Упоминания и уведомления
- По умолчанию для пользователей рекомендуется режим уведомлений `все сообщения`.
- Личное упоминание используйте, когда нужен ответ конкретного человека.
- Массовые упоминания используйте осторожно.
- Так как локальные правила по массовым упоминаниям пока не утверждены, используйте их только когда сообщение действительно важно для всей комнаты.

## Файлы, ссылки и данные
- Ограничений по файлам сейчас не зафиксировано, но это не означает, что можно передавать любые данные без оценки рисков.
- Точных локальных правил по типам данных пока нет.
- До появления отдельного регламента не отправляйте в комнаты чувствительные данные без явной необходимости.
- Если сомневаетесь, можно ли отправлять конкретный файл или сведения, напишите этому боту.

## Внешние пользователи
- Приглашение внешних пользователей не разрешено.
- Если кажется, что для работы нужен внешний участник, не добавляйте его самостоятельно — сначала уточните через этого бота.

## Что делать при ошибочной отправке
Если сообщение ушло не туда:
1. Не игнорируйте это.
2. Оцените, есть ли в сообщении чувствительные данные.
3. Если сообщение не критичное, по ситуации можно отправить корректирующее сообщение в нужную комнату.
4. Если отправлены чувствительные данные, сразу напишите этому боту и укажите:
   - в какую комнату ушло сообщение;
   - что именно было отправлено;
   - успели ли его прочитать;
   - нужны ли срочные действия.

## Когда писать этому боту
Пишите этому боту, если:
- не получается войти;
- не подтверждается новое устройство;
- не читается зашифрованная история;
- видите незнакомую сессию;
- не понимаете, куда писать или какую комнату использовать;
- не уверены, допустимо ли действие по локальным правилам.
