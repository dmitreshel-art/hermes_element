# Эскалация в ИТ и ИБ

## Кратко
- Первая точка обращения по обычным проблемам и по безопасности — этот бот.
- Если вопрос не решается обычной инструкцией, бот передаёт обращение специалистам.
- Для незнакомых устройств, проблем с шифрованием и подозрения на компрометацию не откладывайте обращение.

## Когда писать по обычной поддержке
- не удаётся войти после штатных шагов;
- не приходит ожидаемое подтверждение или вход не завершается;
- не приходят уведомления после базовой проверки;
- не удаётся загрузить файл или использовать штатную функцию клиента;
- не удаётся подтвердить новое устройство;
- не открывается зашифрованная история после стандартных безопасных шагов.

## Контур обработки
- контакт поддержки: этот бот;
- режим работы специалистов: пока не указаны;
- канал эскалации: бот сам передаёт обращение специалистам.

## Когда писать по инциденту безопасности
- подозрение на взлом учётной записи;
- обнаружена незнакомая сессия;
- чувствительные данные отправлены не туда;
- есть признаки компрометации устройства;
- есть вопросы по доступу к зашифрованной истории после рискованных событий.

## Канал по безопасности
- контакт по инцидентам безопасности: этот бот;
- канал ИБ-эскалации: этот бот.

## Что приложить к обращению
- краткое описание проблемы;
- время возникновения;
- тип устройства и ОС;
- версию клиента;
- текст ошибки;
- скриншот, если это уместно;
- что уже было проверено самостоятельно.
