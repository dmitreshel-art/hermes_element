# Recovery key и Secure Backup — локальная памятка

## Кратко
- Recovery key нужен для восстановления доступа к зашифрованной истории.
- Это не пароль от вашей учётной записи.
- Если вы не понимаете, к чему приведёт сброс Secure Backup, не подтверждайте его наугад.
- Если штатное восстановление не получается, напишите этому боту. Если вопрос не решён, бот передаст обращение специалистам.

## Что важно помнить
- Пароль от учётной записи и recovery key — разные вещи.
- Recovery key нужен именно для шифрования и восстановления защищённой истории.
- Если recovery key потерян, часть старой зашифрованной истории может оказаться недоступной.

## Когда это обычно нужно
- при входе с нового устройства;
- при восстановлении доступа к старой зашифрованной истории;
- после переустановки клиента;
- когда Element предлагает восстановить ключи шифрования.

## Безопасный порядок действий
1. Сначала проверьте, есть ли у вас старое активное устройство с уже работающим Element.
2. Если есть старое устройство, сначала попробуйте подтвердить новое устройство через него.
3. Только если Element явно предлагает штатное восстановление через Secure Backup и вы понимаете, что используете правильный recovery key, переходите к восстановлению.
4. Если интерфейс предлагает сбросить backup, остановитесь и ещё раз проверьте, понимаете ли последствия.
5. Если есть сомнения, напишите этому боту до выполнения сброса.

## Чего не делать
- не хранить recovery key в сообщениях, пересылках и общих комнатах;
- не отправлять recovery key этому боту;
- не сбрасывать Secure Backup только потому, что Element показывает предупреждение;
- не путать пароль учётной записи с recovery key.
